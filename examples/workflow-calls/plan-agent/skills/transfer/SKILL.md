# Versatile 请求体组装技能

把用户的一句话拆成有序的原子任务，每个任务调用一次 Versatile 工作流工具，直接传入完整的请求体 JSON：

```json
{"inputs":{"query":"...","intent":"..."}}
```

## inputs 层字段

| 字段 | 说明 | 格式 / 示例 |
|------|------|-------------|
| `query` | **自然语言子任务原文**。第 1 轮直接用业务子任务的自然语言，不要转义成 JSON 字符串。 | `"查余额"` / `"转账5元给李四"` |
| `intent` | 操作意图，固定枚举。 | `"查询账户余额"` / `"快速转账"` |

`wap_userName` 不由本技能填写，由调用方透传。

## 拆分规则

- 用户一句话可能包含多个动作：查余额、给一个或多个人转账。
- 先查余额（若用户提到），再转账。
- 每个「收款人 + 金额」拆成一个独立的转账任务。
- 逐个调用，收到上一个结果后再调下一个。
- 全部完成后，用中文汇总每个任务的结果。

## 调用示例

用户消息：「先查下余额，再给李四和王五各转5元」

拆成 3 个任务，依次调用三次：

1. `{"inputs":{"query":"查余额","intent":"查询账户余额"}}`
2. `{"inputs":{"query":"转账5元给李四","intent":"快速转账"}}`
3. `{"inputs":{"query":"转账5元给王五","intent":"快速转账"}}`

最后汇总：当前余额、给李四转账结果、给王五转账结果。
